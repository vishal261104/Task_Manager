# MongoDB Setup Guide

Your Task Manager application uses **MongoDB** as the database.

## Prerequisites

You have two options:

### Option 1: MongoDB Atlas (Cloud - Recommended)
- Free account at https://cloud.mongodb.com
- No local installation needed
- Accessible from anywhere

### Option 2: Local MongoDB
- Download from https://www.mongodb.com/try/download/community
- Or use package manager:
  - **Mac**: `brew tap mongodb/brew && brew install mongodb-community`
  - **Linux**: `sudo apt-get install mongodb`

## Setup Steps

### 1. Get MongoDB Connection String

**Using MongoDB Atlas (Recommended):**
1. Create free account at https://cloud.mongodb.com
2. Create a new project
3. Create a cluster (M0 free tier)
4. Click "Connect" → "Drivers" → Copy connection string
5. Format: `mongodb+srv://username:password@cluster.mongodb.net/task_manager?retryWrites=true&w=majority`

**Using Local MongoDB:**
```bash
# Start MongoDB
# Windows: MongoDB should auto-start after installation
# Mac: brew services start mongodb-community
# Linux: sudo systemctl start mongod

# Connection string: mongodb://localhost:27017/task_manager
```

### 2. Configure Environment Variables

Create or update `.env` in the Backend directory:

```env
# MongoDB Connection
MONGODB_URI=mongodb+srv://your_username:your_password@your_cluster.mongodb.net/task_manager?retryWrites=true&w=majority

# JWT Secret (keep existing or generate new)
JWT_SECRET=your_jwt_secret_here

# Server Port
PORT=4000

# Environment
NODE_ENV=development

# Optional: timezone used for streak day boundaries
STREAK_TIMEZONE=UTC
```

### 3. Special Characters in Password

If your MongoDB password contains special characters, URL-encode them:
- `@` → `%40`
- `#` → `%23`
- `$` → `%24`
- `%` → `%25`
- `:` → `%3A`

Example:
```
Password: my@pass#123
Encoded: my%40pass%2523123
Connection: mongodb+srv://user:my%40pass%2523123@cluster.mongodb.net/...
```

### 4. Install Dependencies

```bash
cd Backend
npm install
```

### 5. Verify Database Connection

Test the setup:

```bash
npm run setup-db
```

Or start the server:

```bash
npm start
```

Test the health endpoint:
```bash
curl http://localhost:4000/health
```

Should return:
```json
{
  "status": "OK",
  "database": "Connected",
  "timestamp": "..."
}
```

## Collections

The following collections will be auto-created when data is inserted:

- **users** - User accounts with email and passwords
- **tasks** - User tasks with title, description, priority, due date
- **dailyhabits** - Daily habits for tracking
- **useraccount** - Other user-related data (auto-generated by Mongoose)

## Database Management

### View Collections (MongoDB Atlas)
1. Go to https://cloud.mongodb.com
2. Select your cluster → Collections tab
3. Browse collections and documents

### View Collections (Local MongoDB)
```bash
# Connect to mongosh
mongosh

# Use database
use task_manager

# List collections
show collections

# View documents
db.users.find()
db.tasks.find()
db.dailyhabits.find()
```

### Backup Database

**MongoDB Atlas**: Use built-in snapshots

**Local MongoDB**:
```bash
mongodump --uri="mongodb://localhost:27017/task_manager" --out=./backup
```

### Restore Database

```bash
mongorestore --uri="mongodb://localhost:27017/task_manager" ./backup
```

## Troubleshooting

### Connection Error: "bad auth: authentication failed"
- Check username and password in MONGODB_URI
- Verify password special characters are URL-encoded
- For MongoDB Atlas: Check IP whitelist (e.g., allow 0.0.0.0/0 for anywhere)
- Password is case-sensitive!

### Connection Error: "getaddrinfo ENOTFOUND"
- Verify cluster name and region in connection string
- Check internet connectivity
- For local MongoDB: Ensure mongod service is running

### Collections Not Found
- Collections are auto-created on first data insertion
- No manual collection creation needed
- Just use the app normally!

### MongoDB Atlas IP Whitelist
1. Go to Network Access
2. Click "Add IP Address"
3. For development: Add `0.0.0.0/0` (accessible from anywhere)
4. For production: Add your server's IP

## Production Deployment

For cloud deployment (Render, Railway, Heroku, etc.):

1. Use MongoDB Atlas (already cloud-based)
2. Update MONGODB_URI in environment variables
3. Set NODE_ENV=production
4. Keep .env file secure (never commit to git)

### Example Render Deployment

1. Create MongoDB Atlas cluster
2. In Render: Add environment variable MONGODB_URI
3. Deploy your app
4. Test: Visit `https://your-app.onrender.com/health`

## API Remains Unchanged

Even though we switched from PostgreSQL to MongoDB, **all API endpoints work exactly the same**. The database change is transparent to frontend users!
